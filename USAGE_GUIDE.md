# 📚 개선된 페이지 구조 및 사용 가이드

## 🎯 개선 목표

기존의 기능 중심 구조에서 **학습 플로우 중심 구조**로 변경하여 더 효율적이고 자연스러운 학습 경험 제공

## 📊 새로운 페이지 구조

### 1️⃣ **대시보드 (Home - 기본 페이지)**
- **경로:** `/`
- **아이콘:** 📊
- **목적:** 학습 상태를 한눈에 파악

#### 주요 기능:
- **통계 카드 (4개)**
  - 📈 총 질문 수: 등록된 모든 질문의 개수
  - 🎯 학습 진행율: (학습완료 / 전체) × 100%
  - 📌 복습 필요: 아직 배우지 못한 단어 개수
  - ✅ 학습 완료: 학습 완료한 단어 개수

- **빠른 액션 버튼 (3개)**
  - 📝 **테스트 풀기**: 모든 질문으로 테스트
  - 📌 **약점 복습**: 모르는 단어만 복습
  - ➕ **질문 추가**: 새로운 학습자료 등록

- **학습 팁**: 효과적인 학습 방법 제시

#### 사용 흐름:
```
앱 실행 → 대시보드에서 진행률 확인 → 빠른 액션 선택
```

## ⚙️ 최적의 사용 패턴

### 📅 일일 루틴 추천

#### **5분 루틴** (매일 아침)
```
1. 대시보드 접속 → 진행도 확인
2. 약점 복습 → 3~5개 단어 학습
3. 클로징 → 배운 내용 체크
```

#### **20분 루틴** (저녁)
```
1. 대시보드 → 오늘의 목표 확인
2. 테스트 풀기 → 주제 선택 후 10문제
3. 모르는 단어 체크 → 즉시 약점 표시
4. 완료 후 결과 확인 → 통계 기록
```

#### **40분 루틴** (주말)
```
1. 대시보드 → 전체 진행도 확인
2. 새로운 질문 추가 (5~10개)
3. 전체 테스트 → 한 주제 완전 학습
4. 약점 복습 → 모든 모르는 단어 학습
5. 통계 확인 → 주간 진행도 검토
```

---

### 📊 효율성 최적화 팁

#### **학습 단계**
```
1단계: 기초 인식
└─ 질문 둘러보기로 전체 내용 파악
└─ 어려운 부분 표시

2단계: 반복 학습
└─ 테스트 풀기로 이해도 점검
└─ 틀린 문제 모르는 단어 체크

3단계: 집중 복습
└─ 약점 복습으로 부족한 부분 집중
└─ 진행도 80% 이상까지 반복

4단계: 최종 확인
└─ 전체 테스트로 최종 점검
└─ 완료 표시로 마무리
```

#### **주제별 관리 팁**
- 주제는 **의미 있는 단위**로 분류
- 주제별로 **최소 10개 이상**의 질문
- 주 1회 새로운 주제 추가
- 월 1회 전체 주제 정리

#### **모르는 단어 활용법**
- 테스트 중 **즉시** 모르는 단어 표시
- 매일 **짧은 시간에** 약점 복습
- 일주일 이상 학습된 단어는 자동 완료
- 정기적으로 **완료된 단어 정리**

## 🎯 성공 사례

### 예시 1: 영어 단어 학습 (1개월)
```
주제: 영어 > TOEIC 단어
주당 목표: 50개 단어 학습

1주차: 20개 질문 추가 (일 5개씩)
       → 약점 복습 시작
       → 진행도 30%

2주차: 30개 추가 (누적 50개)
       → 테스트 5회 진행
       → 진행도 60%

3주차: 다시 복습 + 20개 추가 (누적 70개)
       → 약점 복습 + 테스트 병행
       → 진행도 80%

4주차: 최종 복습 + 10개 추가 (누적 80개)
       → 전체 테스트 3회
       → 진행도 95% 달성
```

### 예시 2: 시험 대비 (3개월)
```
주제: 수학 > 고등수학 > 미적분
목표: 진행도 100% 달성

월별 진행:
- 1월: 기초 개념 (50개 질문)
- 2월: 심화 문제 (50개 추가)
- 3월: 최종 복습 (모두 학습 완료)
```

## 🚀 시작하기
